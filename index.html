📊 Decisión de Crédito

Decidimos con más datos
y damos opciones al cliente

Nuevas variables del modelo · Doble oferta al cliente · Re-evaluación automática

02
/credit-applications
Modelo más preciso
SCRUM-233
El modelo de decisión se recalibró con nuevos pesos y ahora considera cuota inicial declarada, chasis y papeles del vehículo — datos que antes no se guardaban en el sistema.
Reenvíos que re-evalúan
SCRUM-300
Si el cliente fue rechazado y se reenvía el correo, el modelo vuelve a correr con los datos actualizados. Y la carta enviada se ajusta automáticamente según el estado del crédito.
Doble opción al cliente
SCRUM-41
Cuando el modelo aprueba con margen, el cliente elige entre dos ofertas (por ejemplo, distintos plazos o cuotas iniciales). Su selección queda guardada y se refleja en el simulador.
📊 Impacto — El modelo de riesgo se recalibró con variables que antes no se persistían (chasis, papeles, inicial). El cliente elige entre dos opciones cuando aplica, y el reenvío del correo de aprobación re-evalúa la decisión sin intervención del analista.

/credit-applications
Wizard de ajuste — 3 pasos
1. Editar inputs
Solicitado · plazo · tasa · costo administrativo · cuota inicial
→
2. Preview backend
Endpoint /preview-adjustment devuelve cuota exacta + provisión
→
3. Aplicar & re-evaluar
Modelo se re-corre · cuotas persistidas · email al cliente
Lo que puede hacer el admin
Edita 5 datos del crédito
Monto, plazo, tasa mensual, costo administrativo y cuota inicial
Ve la nueva cuota al instante
El backend calcula y muestra el resultado antes de aplicar
Aplica y el modelo se re-corre
No hace falta pedir nueva validación — todo queda recalculado
Lo que cuida el sistema
Provisión alineada al operador
Redondeada al millar — coincide con la hoja de cálculo manual
Errores claros al admin
Si algo está mal, el sistema explica qué y por qué
Validez de la oferta visible
El admin ve hasta cuándo la nueva oferta sigue siendo válida
🛠️ Impacto — Antes el admin tenía que pedirle a ingeniería ajustar un crédito. Ahora cambia monto, plazo, tasa o cuota inicial desde el panel, ve la nueva cuota al instante, y el modelo se re-evalúa solo.
